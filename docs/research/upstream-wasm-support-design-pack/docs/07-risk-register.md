# Risk Register

| Risk                                        | Consequence                                                                                                 | Mitigation                                                                                                                           | Owner / gate                  |
| ------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------- |
| Native default regression                   | Existing native users lose a codec, TLS provider, filesystem path, or performance characteristic.           | Preserve public feature lists; compare native feature graphs; run default and curated all-feature tests before each release.         | Owning crate / native CI      |
| SemVer change in feature composition        | Consumers rely on a transitive dependency or capability removed from a public feature.                      | Keep logical features and native composition in the compatibility PR; review cleanup as a separate change with release notes.        | Maintainers / feature diff    |
| Feature unification reintroduces a backend  | A consumer enables `zstd` or defaults through another edge and breaks WASM.                                 | Put implementation dependencies in target tables; test explicit feature-on profiles; inspect reverse feature paths.                  | Arrow, Parquet / graph CI     |
| Target predicate drifts                     | A change intended for `wasm32-unknown-unknown` affects WASI, Emscripten, or wasm64 without tests.           | Use `all(target_arch = "wasm32", target_os = "unknown")`; broaden only through a named policy and added targets.                     | Each repository / cfg review  |
| Existing C-backed WASM users lose behavior  | A configured build that used `zstd-sys` stops offering zstd.                                                | Decide whether that path is unsupported or deserves an advanced opt-in; publish the decision in release notes.                       | Arrow maintainers             |
| Unsupported-codec ambiguity                 | Users assume `feature = "zstd"` guarantees a backend on every target.                                       | Document logical enablement versus target availability; return operation, codec, and target in the error.                            | Arrow, Parquet / error tests  |
| Late writer failure                         | An output stream contains partial data before codec failure.                                                | Instantiate or validate the codec before committing compressed output; add sink-state tests.                                         | Arrow, Parquet                |
| Metadata inspection regresses               | A reader rejects a file before the application asks for compressed data.                                    | Keep codec construction lazy enough for footer and schema inspection; add fixed-file tests.                                          | Parquet, Arrow IPC            |
| Pure-Rust zstd incompatibility              | A decoder accepts malformed data, rejects common frames, or mishandles dictionaries and checksums.          | Require golden corpora, malformed-input tests, memory limits, fuzzing, checksum coverage, and dictionary policy.                     | Follow-up codec PR            |
| Pure-Rust zstd performance                  | Browser query latency exceeds product budgets.                                                              | Benchmark representative pages and batches against native zstd; keep adoption conditional on a published budget.                     | Follow-up codec PR            |
| Browser binary growth                       | New codec or host dependencies make the WASM artifact too large.                                            | Record raw and compressed size deltas by feature; keep zstd decode optional through the logical feature; avoid duplicate JS bridges. | Browser CI                    |
| Lockfile and registry drift                 | A source SHA no longer reproduces the transitive graph.                                                     | Record lock hash, use `--locked`, and run a separate newest-compatible job.                                                          | Research and release CI       |
| Dev-dependency masking                      | Repository tests compile because a dev dependency enables `wasm_js` or another backend that consumers lack. | Keep isolated consumer fixtures with their own lock and no workspace dev dependencies.                                               | `object_store`, DataFusion    |
| No-Clang false confidence                   | Native `-sys` crates stay in the graph but avoid a failing build path in one image.                         | Pair the no-Clang image with target-filtered active normal/build graph assertions.                                                   | All owning crates             |
| `http-base` retains hidden entropy          | A host-neutral consumer still fails through `rand -> getrandom`.                                            | Remove default OS-seeded jitter; inject RNG only from native or `web` host profiles.                                                 | `object_store`                |
| Browser timer or executor assumption        | Retries or multipart work panic under a browser executor.                                                   | Test JS timer and local scheduling in a real browser; keep retry and multipart changes separate.                                     | `object_store`                |
| Browser range response buffers whole object | A server ignores Range and the browser consumes a large Parquet object.                                     | Reject ordinary non-full `200` responses; allow full-body fallback only under an explicit byte bound.                                | `object_store` protocol suite |
| Object changes between ranges               | The reader combines bytes from different object versions.                                                   | Use strong ETag validation; treat `If-Range` mismatch as identity invalidation and replan.                                           | HTTP adapter                  |
| CORS hides the cause                        | Browser reports a generic network error, leading to a false diagnosis.                                      | Emit a qualified CORS or network-policy diagnostic; do not name a missing header without evidence.                                   | HTTP adapter                  |
| Server returns encoded ranged data          | Byte offsets apply to another representation.                                                               | Rely on Fetch's identity request and reject any observed non-identity ranged response.                                               | HTTP adapter                  |
| Crypto provider leaks into generic HTTP     | AWS-LC or `ring` increases toolchain, entropy, and bundle exposure.                                         | Target-gate the direct provider dependency; deny providers in the generic browser fixture.                                           | `object_store` graph CI       |
| Filesystem code appears available           | A compile success is read as browser filesystem support.                                                    | Keep `fs` inactive in browser profiles and return `NotSupported` at construction or first use.                                       | `object_store`, DataFusion    |
| DataFusion compile/runtime conflation       | A default WASM check is presented as support for every execution plan.                                      | Publish the initial no-disk, one-partition runtime tier and tie claims to real-browser tests.                                        | DataFusion                    |
| DataFusion capability drift                 | A new Parquet default enters DataFusion without review.                                                     | Disable dependency defaults and maintain the explicit capability list in the direct manifests.                                       | DataFusion                    |
| Kernel fork drift                           | delta-rs results cannot be reproduced from the named upstream kernel revision.                              | Pin the Buoyant revision and keep a separate upstream-patched smoke.                                                                 | delta-rs                      |
| Entropy abstraction grows too early         | Kernel APIs gain a public capability used by one call site.                                                 | Classify random uses first; gate write paths or reuse engine services before adding an API.                                          | delta-kernel                  |
| Native engine accumulates WASM exceptions   | The default engine becomes hard to maintain and still carries native dependencies.                          | Keep a separate browser engine that implements core engine traits.                                                                   | delta-kernel, delta-rs        |
| Downstream policy leaks upstream            | Arrow or `object_store` gains Axon, CDN, or delta-rs assumptions.                                           | Keep upstream contracts at codec, HTTP service, runtime service, and engine seams.                                                   | Architecture review           |

## Risk Acceptance

The first PR stack accepts two visible limitations:

- zstd files compile into browser applications but return a target-unavailable error when code asks
  for zstd data;
- the DataFusion browser tier excludes disk, spill, broad task graphs, and untested codecs.

Those limits are part of the published support contract. Silent build failure, panic, whole-object
buffering, and mixed-version range assembly are not accepted.
